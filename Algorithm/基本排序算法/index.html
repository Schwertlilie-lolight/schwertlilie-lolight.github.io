基本排序算法

发表于 2017-02-05 | 更新于: 2017-12-30 | 分类于 Algorithm |

成本模型：
一般为元素比较和交换的次数。
如果不需要比较和交换，那么我们计算数组访问的次数

1. 选择排序

  1. 找到最小元素
  2. 把它和第一个元素相交换
  3. 在剩下的元素中寻找最小元素
  4. 将其和第二个元素交换
  5. 重复以上步骤，直到数组排序完毕（元素指针走到数组末尾）

1.1 性能

在一个长度为 NNN 的数组中，它需要 N2/2{N^2}/{2}N​2​​/2 次的比较和 NNN 次的交换

特点：

  1. 所需时间和输入模型无关
  2. 数据的移动是最少的

2. 插入排序

插入排序的主要思想是在数组的无序部分取元素插入到有序部分中，从而逐步构建有序。

举一个按照升序排列的例子

  1. 将 a[i]a[i]a[i] 和 a[0]a[0]a[0] 到 a[i−1]a[i - 1]a[i−1] 中所有比它小的元素依次交换
  2. 保证在 a[i]a[i]a[i] 的左边，元素总是有序的
  3. 当 iii 指针到达数组末尾的时候，排序就完成了。

2.2 性能

此算法的性能和输入有关。
如果输入的序列已经是部分排序的，那么使用这个算法将会比较快；
但是由于存在嵌套的 for 循环，在最坏情况下仍然需要 N2N^2N​2​​ 次交换

2.3 优化

2.3.1 使用移动代替交换

可以在上述算法实现上进行进一步调优。
可以简单的将大的元素往右移动，从而空出一个正确的位置，将所需元素插入即可；
而不是每次都要交换一次元素。
这个调优将能节省一半的数组访问开支

2.3.2 使用二分查找

由于是在有序部分寻找恰当位置插入，可以使用二分查找提高搜索效率

3. 希尔排序

希尔排序是基于插入排序的一种排序算法。
其基本思想是让元素在 hhh 步长中有序
如果 hhh 很大，那么我们就可以将一个元素一次性移动很远

希尔排序比插入排序和选择排序都要快；
